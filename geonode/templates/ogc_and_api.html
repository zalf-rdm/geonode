{% trans "OGC Services and API of the ZALF Repository" %}

{% trans "The ZALF Repository provides standardized access to geospatial data through OGC-compliant web services and modern Web APIs. These interfaces enable integration with GIS software, web applications, and automated workflows following FAIR and research data management principles." %}

{% trans "1. Overview" %}

{% trans "All published geospatial layers in the ZALF Repository can be accessed through Open Geospatial Consortium (OGC) services. Depending on the dataset type, different service interfaces are available, such as WMS for visualization, WFS for vector data access, and OGC API for modern web-based access." %}

  • {% trans "Map visualization for web and desktop GIS (e.g. QGIS, WebGIS clients)." %}
  • {% trans "Download and filter of vector features and coverage data." %}
  • {% trans "Integration into external portals, dashboards, or analysis workflows." %}

{% trans "2. Classic OGC Web Services" %}

{% trans "The repository exposes datasets via standard OGC Web Services. Typical endpoints include WMS, WFS, WCS and related capabilities documents." %}

  • WMS (Web Map Service) – {% trans "for map visualization, styled raster and vector layers, and background maps." %}
  • WFS (Web Feature Service) – {% trans "for access to vector features (points, lines, polygons) in machine-readable formats such as GML or GeoJSON." %}
  • WCS (Web Coverage Service) – {% trans "for raster and coverage data, including gridded environmental variables or model outputs." %}
  • WMTS / Tiled Services – {% trans "for efficient, tiled map rendering in web applications." %}

{% trans "The capabilities of each service describe the available layers, coordinate reference systems, and supported formats." %}

{% trans "3. OGC API – Modern Web APIs" %}

{% trans "Where supported by the platform, the ZALF Repository also offers OGC API endpoints (e.g. OGC API - Features) that provide RESTful, JSON-based access to geospatial data." %}

  • {% trans "Simple HTTP/HTTPS requests with JSON responses." %}
  • {% trans "Filtering, paging, and querying of features via URL parameters." %}
  • {% trans "Easy integration into web applications, scripts, and data science workflows." %}

{% trans "OGC API services complement the classic WMS/WFS protocols and are particularly suitable for lightweight clients and programmatic access." %}

{% trans "4. Example Service Endpoints" %}

{% blocktrans %}The exact URLs of the services may depend on the deployment, but commonly follow patterns like:{% endblocktrans %}

  • WMS: https://repository.zalf.de/geoserver/ows?service=WMS&request=GetCapabilities
  • WFS: https://repository.zalf.de/geoserver/ows?service=WFS&request=GetCapabilities
  • WCS: https://repository.zalf.de/geoserver/ows?service=WCS&request=GetCapabilities
  • OGC API - Features (example): https://repository.zalf.de/oapi/collections/<collection-id>/items

{% trans "For each dataset, direct service links are typically available on the resource detail page within the repository interface (e.g. under “Download”, “Services” or “API”)." %}

{% trans "5. Using the Services in GIS Software" %}

{% trans "You can connect to the ZALF Repository from standard GIS tools such as QGIS or ArcGIS by adding the corresponding OGC service URL." %}

  • {% trans "In QGIS: use ‘Add WMS/WMTS Layer’ or ‘Add WFS Layer’ and paste the service URL." %}
  • {% trans "Select the desired layer from the capabilities list and add it to your map." %}
  • {% trans "For OGC API, use the appropriate plugins or HTTP-based data connectors where available." %}

{% trans "Depending on the access rights of the dataset, authentication with your repository account may be required." %}

{% trans "6. Authentication and Access Control" %}

{% trans "Some services and layers are publicly available, while others require authentication or are restricted to specific project partners." %}

  • {% trans "Public datasets: can be accessed without a login, depending on the license and visibility settings." %}
  • {% trans "Restricted datasets: may require login credentials or explicit authorization from the data owner." %}
  • {% trans "Internal datasets: are limited to ZALF or project-internal users and are not accessible externally." %}

{% trans "The access level is defined by the dataset owner during publication and follows the Data Policy of the ZALF Repository." %}

{% trans "7. Formats, CRS and Interoperability" %}

{% trans "The ZALF Repository aims to support interoperable, standards-based formats and coordinate reference systems (CRS)." %}

  • {% trans "Common output formats include PNG, JPEG, GeoTIFF, GML, GeoJSON, and CSV (where applicable)." %}
  • {% trans "Typical coordinate reference systems include EPSG:4326 and national or project-specific CRSs where required." %}
  • {% trans "Metadata describe the spatial reference, resolution, and constraints for each dataset." %}

{% trans "8. Rate Limits and Good Practice" %}

{% trans "To ensure stable operation for all users, please follow these guidelines when accessing OGC services and APIs:" %}

  • {% trans "Avoid extremely frequent automated requests without caching." %}
  • {% trans "Use spatial and temporal filters to limit the requested extent and data volume." %}
  • {% trans "Prefer downloading datasets for heavy analysis instead of repeatedly querying map services." %}

{% trans "If you plan to build large-scale or automated services based on the repository, please contact the support team in advance." %}

{% trans "9. Support and Contact" %}

{% trans "For questions related to OGC services, API usage, or technical integration, please contact the ZALF research data management or repository support team." %}

{% trans "We can provide additional technical details, example configurations (e.g. for QGIS), and advice on integrating the repository services into your workflows." %}
