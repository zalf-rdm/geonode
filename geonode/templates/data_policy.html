{% trans "Data Policy of the ZALF Repository" %}

{% trans "This Data Policy describes the principles, responsibilities, and rules that govern the submission, storage, publication, and reuse of datasets in the ZALF Repository. The policy follows international best practices for Research Data Management (RDM) and the FAIR principles (Findable, Accessible, Interoperable, Reusable)." %}

{% trans "1. Purpose and Scope" %}

{% trans "The ZALF Repository provides a central platform for sharing, preserving, and managing research data generated within ZALF or associated projects. Its purpose is to ensure long-term accessibility, reproducibility, and quality of research outputs." %}

  • {% trans "This policy applies to all datasets uploaded, curated, or published through the repository." %}
  • {% trans "It concerns both public datasets and those restricted for internal or controlled access." %}
  • {% trans "It covers metadata, documentation, and any associated supplementary material." %}

{% trans "2. Responsibilities of Data Contributors" %}

{% trans "Individuals who upload data to the repository must ensure that:" %}

  • {% trans "They have the right to publish and share the submitted data." %}
  • {% trans "The dataset does not contain confidential, personal, or sensitive information unless explicit permission exists." %}
  • {% trans "The data and metadata are accurate, complete, and comply with ethical and legal requirements." %}
  • {% trans "All relevant documentation, methods, and descriptions are included so that others can understand and reuse the data." %}
  • {% trans "Licensing and access conditions are correctly defined during upload." %}

{% trans "3. Licensing and Access Levels" %}

{% trans "Datasets in the ZALF Repository must be assigned a license indicating how they may be accessed and reused. Contributors may select from a range of open and restricted options." %}

  • {% trans "Open Access" %} – {% trans "Data may be freely accessed and reused under an openly accepted license (e.g., CC-BY, CC0)." %}
  • {% trans "Restricted Access" %} – {% trans "Data may require approval before use or may be limited to specific groups or collaborators." %}
  • {% trans "Internal Access" %} – {% trans "Data are accessible only within ZALF or associated project teams." %}

{% trans "Contributors are responsible for selecting a license that reflects legal obligations, project agreements, and intended reuse." %}

{% trans "4. Data Quality and Metadata Standards" %}

{% trans "All datasets must be accompanied by sufficient metadata to ensure discoverability and reuse. The repository encourages the use of standardized formats, controlled vocabularies, and consistent documentation." %}

  • {% trans "Titles, descriptions, and keywords must clearly represent the content." %}
  • {% trans "Spatial, temporal, and methodological information must be provided when applicable." %}
  • {% trans "Datasets should follow discipline-specific standards where available (e.g., ISO, INSPIRE, domain ontologies)." %}
  • {% trans "Quality assurance steps, limitations, and uncertainties should be documented transparently." %}

{% trans "5. Data Preservation and Retention" %}

{% trans "The repository ensures long-term preservation of datasets using stable storage infrastructure and versioning mechanisms. Once published, datasets are maintained for long-term access unless legally required to be removed." %}

  • {% trans "Published datasets receive persistent identifiers to guarantee long-term referencing." %}
  • {% trans "Updated versions can be uploaded, but previous versions remain archived for reproducibility." %}
  • {% trans "Files may only be removed in cases of legal, ethical, or documented scientific errors." %}

{% trans "6. Ethical and Legal Compliance" %}

{% trans "All data contributors must ensure compliance with ethical standards, data protection regulations, and contractual obligations. Sensitive datasets may require anonymization or restricted sharing conditions." %}

  • {% trans "Personal data must not be uploaded unless fully anonymized or explicitly permitted by applicable regulations." %}
  • {% trans "Data derived from third parties must respect original licensing terms and copyright." %}
  • {% trans "Ethical approval must be documented where relevant." %}

{% trans "7. Reuse of Repository Data" %}

{% trans "Users must follow the specified license of each dataset when downloading or reusing data. Proper citation of the dataset and its contributors is mandatory." %}

  • {% trans "Citations should include the dataset title, authors, publication year, repository name, and persistent identifier." %}
  • {% trans "Reuse for commercial or derivative purposes must respect license-specific conditions." %}

{% trans "8. Changes to this Policy" %}

{% trans "The Data Policy may be updated over time to reflect evolving standards, legal requirements, and institutional guidelines. Significant changes will be communicated to users." %}

{% trans "Contact" %}

{% trans "For questions related to data publication, licensing, or RDM practices, please contact the ZALF research data management support team." %}
