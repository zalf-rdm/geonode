{% trans "Upload Data to the ZALF Repository" %}

{% trans "This page provides general instructions on how to upload research data to the ZALF Repository using the Upload Tool. The process follows FAIR principles (Findable, Accessible, Interoperable, Reusable) and Research Data Management best practices to ensure high-quality, well-documented, and sustainable datasets." %}

{% trans "1. Before You Start" %}

{% trans "Before uploading, please ensure that your dataset is complete, well-structured, and accompanied by clear documentation. Preparing your data in advance will improve discoverability and reuse." %}

  • {% trans "Organize your files logically (e.g., by variables, years, sampling locations)." %}
  • {% trans "Include readme files, codebooks, or data dictionaries when relevant." %}
  • {% trans "Remove confidential or personal information unless explicit approval exists." %}
  • {% trans "Ensure you have the right to publish and share the data." %}

{% trans "2. Using the Upload Tool" %}

{% trans "The Upload Tool assists you in adding datasets to the ZALF Repository in a structured and guided way. You will be asked to provide metadata, describe your data, and assign licenses and access permissions." %}

  1. {% trans "Select Your Files" %} — {% trans "Upload individual files or compressed folders (ZIP). Large datasets may require additional steps depending on file size." %}
  2. {% trans "Describe Your Data" %} — {% trans "Fill in the metadata form including title, description, spatial and temporal coverage, keywords, and responsible institutions." %}
  3. {% trans "Choose License and Access Level" %} — {% trans "Decide how others may access or reuse your dataset (open, restricted, or internal only)." %}
  4. {% trans "Validate" %} — {% trans "The Upload Tool will check file formats, metadata completeness, and required information." %}
  5. {% trans "Submit" %} — {% trans "After submission, your dataset is processed and becomes available in the catalogue depending on the selected publication settings." %}

{% trans "3. FAIR Data Principles" %}

{% trans "When uploading data, please consider the following FAIR principles:" %}

  • {% trans "Findable:" %} {% trans "Provide clear metadata, stable identifiers, keywords, and meaningful titles." %}
  • {% trans "Accessible:" %} {% trans "Select an appropriate license and define accessibility conditions." %}
  • {% trans "Interoperable:" %} {% trans "Use standard formats where possible (CSV, GeoTIFF, NetCDF, Shapefile, etc.)." %}
  • {% trans "Reusable:" %} {% trans "Offer enough context, documentation, and provenance information for third-party reuse." %}

{% trans "4. Metadata Guidelines" %}

{% trans "Good metadata is essential for proper discovery and reuse. The Upload Tool will guide you, but consider including additional information whenever possible." %}

  • {% trans "Methods and protocols used to collect or produce the data." %}
  • {% trans "Units, abbreviations, and variable descriptions." %}
  • {% trans "Spatial reference systems and coordinate information." %}
  • {% trans "Instrument names, calibration details, or model configurations." %}
  • {% trans "Known limitations, uncertainties, or data quality notes." %}

{% trans "5. After Uploading" %}

{% trans "Once your dataset is successfully uploaded, you can edit its metadata, update files, or adjust permissions. Each dataset receives a unique identifier and becomes part of the ZALF Repository catalogue." %}

  • {% trans "Review your dataset page to confirm that all information is correct." %}
  • {% trans "If needed, add additional documentation or supporting material." %}
  • {% trans "Share the permanent link with collaborators or project partners." %}

{% trans "Need Help?" %}

{% trans "If you require support with the upload process, metadata preparation, or RDM best practices, please contact the ZALF research data management team." %}
